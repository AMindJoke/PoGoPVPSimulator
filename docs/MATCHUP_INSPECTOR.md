# Matchup Inspector

The Matchup Inspector is the first small user-facing slice of the analysis layer. It sits between raw matchup simulation results and future Battle Coach features.

## Goal

The inspector explains the current 1v1 matchup without changing the battle engine:

- who wins the even-shield states;
- how close the matchup is;
- whether shield count changes the result;
- whether a concrete flip line already exists;
- how to open a specific shield scenario in the battle view.

## Data Flow

1. The simulator builds the normal matchup matrix.
2. The inspector reads the same cached matrix cells for 0-0, 1-1, and 2-2 shields.
3. If a cell is already available, it is shown as precomputed.
4. If a cell needs to be generated by the browser, it is treated as live simulation data.
5. The inspector converts those cells into a deterministic view model.

The current implementation does not invent alternate lines. Preview buttons appear only when the matrix already has a concrete flip suggestion.

## Current MVP

The MVP is intentionally small:

- matchup header with Pokemon, sprites, league, CP cap, moves, and IV profile;
- even-shield summary for 0-0, 1-1, and 2-2;
- reliable flags such as close result, shield dependency, bait sensitivity, and potential flip;
- Open in Battle per even-shield scenario;
- Preview only for existing matrix flip suggestions.

The Inspector also displays Matchup Story above the raw even-shield overview. Matchup Story provides the concise explanation, while the existing scenario rows remain the detailed and directly actionable view. See `MATCHUP_STORY.md` for its signals, confidence rules, and template architecture.

## Orientation

The analysis module supports A/B orientation and can invert cached results when only the reverse matchup is available. This keeps future offline tooling reusable for both matrix and coach-style explanations.

## Future Work
